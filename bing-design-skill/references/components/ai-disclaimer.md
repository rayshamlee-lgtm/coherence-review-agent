# AI Disclaimer

**Figma Node ID:** `31812:73607` · **Type:** `COMPONENT_SET` · **Group:** Feedback · **Tier:** Base / Foundational Atom · **Variants:** 2 · **Consumed by:** AI-generated answer cards, Copilot responses, AI summaries, chat interfaces

## Summary

The AI Disclaimer displays a notice indicating that content was generated by AI. It provides transparency about content origin and includes a hover state for link interaction, allowing users to learn more about AI-generated content policies.

This component appears below or alongside any AI-generated content (summaries, answers, suggestions), in Copilot response areas, and wherever regulatory or policy requirements mandate AI content disclosure. It is not for general informational notices (use Toast Neutral), error or warning messages (use Toast with semantic style), or sponsored content labeling (use Sponsored Slug).

---

## Variants / States

| State | Description |
|-------|-------------|
| **Rest** (default) | Static display; flyout is hidden |
| **Hover** | Info icon is hovered; flyout is revealed |

---

## Anatomy

1. **Info icon** — InfoSparkle icon at 16×16px centered within a 28×28px container, matching icon button sizing
2. **Flyout** — Reuses the `SerpFlyout` component with `pointingDirection="down"`, `beakAlignment="end"`, `triggerMode="hover"`
3. **Disclaimer text** — Short notice inside the flyout bubble that the content is AI-generated

---

## Visual States

| State | Visual Treatment |
|-------|-----------------|
| Rest | Icon only; flyout hidden |
| Hover | Flyout bubble visible with disclaimer text and interactive link |

The hover zone extends continuously from the icon to the flyout bubble. Setting `state="hover"` force-shows the flyout (used for static design previews). The flyout also appears on keyboard focus for accessibility, and hides when the cursor leaves the icon and bubble area.

---

## Usage Rules

- This component is placed directly top-right of AI-generated content.
- Spacing between the AI content and the disclaimer: `x-small`.
- Disclaimer text is brief and factual (e.g., "AI-generated · Learn more"). It is clear and non-alarming.
- The hover interaction leads to more information about AI content policies.
- The disclaimer is kept visually subordinate to the content it describes.
- Every AI-generated content block includes this component — no exceptions.

---

## Do / Don't

### Do
- Display the disclaimer on all AI-generated content without exception.
- Keep the disclaimer text brief and factual.
- Ensure the hover state provides a clear path to more information.

### Don't
- Don't hide or obscure the disclaimer — transparency is the core purpose.
- Don't style the disclaimer to look like an error or warning.
- Don't use for non-AI content labeling — use Sponsored Slug for ads.

---

## Dark Mode Notes

The disclaimer link must meet WCAG AA contrast requirements against its background in both light and dark themes. Token-based color application handles theme switching automatically.
